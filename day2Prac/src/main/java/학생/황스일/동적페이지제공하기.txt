
동적페이지만들기(프로그램- 자바 - 상속에서 만드는 자바 프로그램 - httpServlet)
1.httpServlet 상속받은 클래스 작성하기
2.약속된 메서드(serivce) 즉, 서비스할 내용을 오버라이드하기
3.서비스할 비지니스 로직을 별도의 자바 파일로 분리하기 
4.서블릿 등록하기,  url-pattern 지정하기 - 중복되지 않기


웹애플리케이션

웹서버 - (응답) - 클라이언트(요청)

httpServlectReqest (요청에 관련된 정보)
HttpServlectResponse (응답에 관련된 정보)

요청                    응답
-Get                   -Get(서버가 가지고 있는 정보를 단지 제공해 주는 서비스)
-Post                  -Post

요청을 구분하는 예시
get  : a태그, url직접요청, from태그 method="get" => 서버는 get요청을 인식
post : form 태그 method="post"              => 서버는 post요청으로 인식

get요청: 사용자가 보내느 데이터를 헤더에 담는다. url노출, 데이터방식, 크기가 정해져있다
post요청 : 대표적인(회원가입) 사용자가 보내는 데이털를 요청 바디에 담는다. 노출되지 않음, 데이터형식, 크기가 제한이 없다

요청을 구분하는 이유
효율적인 서버의 응대
get : 가벼운 요청
post: 무거운 요청

serivce - > request요청 정ㅇ보를 확인 -> get -> doGet, post => doPost 요청해 줌
요청을 구분해서 요청을 처리하는게 일반적임 !!!!!
